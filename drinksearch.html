drinksearch

Customers scan a bar code, and are sent to drinksearch where they can choose from up to 100 drinks from the data base, and see which ingredients are used as well as how they're made.

View app

Objective

The objective for this project was to make use of an extensive data base of information, using an api to fetch this data and append specific information from these objects to the DOM. I wanted to keep this web app nice and simple to use, all created on a single HTML file and with a minimal UI.

Techs used

I decided to stick with vanilla HTML, CSS and JavaScript for this project as I felt there was no need to bring in any frameworks or libraries, plus I like a challange.

The project

To begin this project, I had a design in mind and got started with implementing the functionality. My plan was to have a multiple search option in the navigation bar, options include: Dynamic search bar, A-Z, Spirits and random drink selection.

Once a value is entered, a list is then appended using literal HTML blocks in JavaScript. To make these newly created informational cards appear above the hero section, I utilised the CSS property z-index heavily. This was also replicated for the modal when clicking on these cards, a closer look at the selected item will appear, but with additional information.

Issues

I ran into a few issues with the API, to make use of the information needed, I first needed to make a search for the specific ID, and then make an additional search using the ID as a parameter. Other issues mostly included CSS as everything is essentially floating above one another. Mobile soft keyboard needed to be manually closed on scroll, I used JavaScript to achieve this as well as setting a timeout on the dynamic search to avoid issues when typing fast.

Designing the end product

For the design, I was inspired by the iOS Notification Center, the glass above a blurred background seemed like a fitting model. I then did some research and landed on the Mojito as the main attraction for this app. And with that in mind, I designed this fully responsive app using Figma.
